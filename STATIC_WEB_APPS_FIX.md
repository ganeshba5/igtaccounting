1. **Go to GitHub**: https://github.com/ganeshba5/igtaccounting/settings/secrets/actions

2. **Find or create** secret: `AZURE_STATIC_WEB_APPS_API_TOKEN`

3. **Set the value to**:

5. **Retry the deployment**:

   - Re-run the failed workflow

### Option 2: Use Auto-Generated Workflow Instead

I noticed you have TWO Static Web Apps workflows:

The auto-generated one might already have the correct token set up. You could:

1. **Delete** the manual workflow:
   ```bash

   git commit -m "Remove manual Static Web Apps workflow, use auto-generated one"
   git push origin main
   ```

2. **Or** update the manual workflow to match the auto-generated one's configuration
